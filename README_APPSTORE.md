## アプリ説明文（日本語）

Glimpseは、パスコードで守られたプライベートな写真保管庫です。大切な写真を、カメラロールとは完全に切り離された安全な空間に保管できます。

**主な機能**

- パスコードによるロック — 4桁のパスコードでアプリへのアクセスを保護します。10回連続で失敗すると、すべてのデータが自動消去されます。
- アルバム管理 — 複数のアルバムを作成し、写真を整理できます。サブアルバムの作成や並び替えにも対応しています。
- 写真の取り込み — カメラロールから写真を選択してインポートします。インポートされた写真はアプリ内に独立して保存されます。
- ピンチズームで閲覧 — 写真を直感的な操作で拡大縮小して閲覧できます。
- お気に入り管理 — 大切な写真にハートマークをつけて素早くアクセスできます。
- 共有機能 — 必要なときは標準の共有シートから写真を共有できます。

**セキュリティの特徴**

アプリをバックグラウンドから復帰したとき、必ずロック画面に戻ります。アプリスイッチャーでは画面の内容が隠れるため、のぞき見を防ぎます。写真データはiCloudバックアップの対象外の領域に保存されます。iTunesやFinderのファイル共有でもデータは見えません。

シンプルなデザインで、必要な機能だけを備えた写真保管庫です。

## プライバシーポリシー要点

### データの収集と利用

- **収集するデータ**: なし。Glimpseはユーザーの個人情報・位置情報・使用統計を一切収集しません。
- **外部送信**: ネットワーク通信は行いません。すべてのデータはデバイス内にのみ保存されます。
- **サードパーティ共有**: データをいかなる第三者とも共有しません。

### データの保存

- **写真データ**: デバイス内のアプリ専用領域（Caches）に保存。iCloudバックアップ対象外。
- **パスコード**: iOS Keychain（暗号化済みセキュアストレージ）に保存。
- **データベース**: アプリのサンドボックス内にのみ存在。他のアプリからアクセス不可。

### ユーザーの権利

- アプリを削除することで、すべてのデータが完全に消去されます。
- パスコードの連続誤入力（10回）でもすべてのデータが消去されます。

### 使用する権限

| 権限 | 用途 |
|------|------|
| フォトライブラリ（読み取り） | 写真のインポート時のみ使用 |

## App Store審査メモ

- ネットワーク通信なし（審査時にオフライン動作確認可能）
- フォトライブラリへのアクセスはインポート操作時のみ
- パスコード機能はローカルのみで動作、Apple IDや生体認証とは独立
- 暗号化技術の使用: iOS標準Keychain APIのみ（輸出規制申告: 標準暗号化）
